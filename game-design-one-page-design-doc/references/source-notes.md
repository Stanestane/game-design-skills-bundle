# Source Notes

Distilled from:
- Game Dev Underground one-page game design document template
- GDC Vault overview for **One-Page Designs**

## Core takeaways

### 1. One page is a communication constraint
The point is not novelty. The point is forcing clarity and brevity.

### 2. People skim
The GDC overview makes the blunt point: if people often only read the first page, then make the first page do the real work.

### 3. A one-pager should carry the idea, not the whole production bible
It should communicate:
- what the game is
- what makes it distinct
- what kind of experience it promises
- enough implementation direction to align the team

### 4. Use explicit sections
The template suggests this compact structure:
- Game Identity / Mantra
- Design Pillars
- Genre / Story / Mechanics Summary
- Features
- Interface
- Art Style
- Music / Sound
- Development Roadmap / Launch Criteria

### 5. Good one-pagers trade completeness for readability
Omit detail that does not help the reader grasp the design quickly.

### 6. Production framing still matters
A one-page design doc is not just creative fluff. Platform, audience, milestones, and launch criteria help make it actionable.

## Practical guidance for this skill

- Make the mantra visible early.
- Keep pillars few and sharp.
- Use a short summary paragraph.
- Keep features selective.
- Treat art and sound as directional references, not exhaustive style bibles.
- Keep roadmap milestones coarse and readable.
- If the page is crowded, rewrite shorter; do not shrink forever.
